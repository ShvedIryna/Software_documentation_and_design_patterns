## Лабораторна робота №2: Трьохрівнева архітектура та ORM
**Завдання:** Реалізація серверної частини застосунку з розділенням на рівні відповідальності.

### Структура проекту:
1.  **Рівень доступу до даних (Data Access Layer):** * Використано ORM **SQLAlchemy** для роботи з SQLite.
    * Реалізовано зчитування даних з `.csv` файлу через `CSVReader`.
    * Зв'язок реалізовано через інтерфейс `IFinancialRepository` для дотримання принципу інверсії залежностей.
2.  **Рівень бізнес-логіки (Business Logic Layer):**
    * Клас `FinancialManager` координує вичитку файлу, створення моделей та збереження їх у базу даних.
3.  **Презентаційний рівень:** * Представлений інтерфейсом `IFinancialApp`.
4.  **Утиліти:**
    * Створено скрипт `generate_csv.py` для генерації тестових даних (1000+ рядків).

## Лабораторна робота №3: Веб-додаток на базі MVC
**Завдання:** Створення веб-інтерфейсу для візуалізації та керування фінансовими даними за допомогою шаблону MVC.

### Реалізація:
* **Model:** Використання існуючих моделей `FinancialData` та логіки взаємодії з БД через SQLAlchemy.
* **View:** Реалізовано за допомогою HTML-сторінок (Jinja2 шаблони) з використанням Bootstrap 5 для адаптивного дизайну.
* **Controller:** Flask-маршрути в `app.py`, що обробляють запити користувача (відображення сторінки, додавання, редагування та видалення).

### Ключові функції (CRUD):
1.  **Візуалізація:** Виведення історії транзакцій у вигляді таблиці з пагінацією по 10 записів на сторінку.
2.  **Додавання:** Модальне вікно для створення нових фінансових записів.
3.  **Редагування:** Можливість оновлення існуючих даних через модальне вікно безпосередньо зі списку.
4.  **Видалення:** Функція видалення записів з підтвердженням дії.
5.  **Бізнес-логіка:** Розрахунок чистого прибутку для кожної операції та вичитка даних через рівень бізнес-логіки.
